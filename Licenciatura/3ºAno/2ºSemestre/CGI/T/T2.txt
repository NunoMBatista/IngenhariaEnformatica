Lets say we want to translate a cube into space... using matrixes.
Lets fetch each vertix and define their coordinates, (x, y, z).
So, for a cube we will have a N*3 matrix (n the number of vertices, and 3 the 3D space).

We need to trnalsate this into a screen.
The screen is  a 2D surface !

So we need;
(N*3) * P = (N*2)

The P is a "magical" thing that can make the translation into a 2D surface (still have N point, but now translated into 2 dimentions!

P(4x4)
Note that this matrix that we use will always be a 4*4 matrix.
Let's try and see why.

Lets take a transformation:
x' = x
y' = y
z' = z

If i stay still this relation will maintain.
Let's say i want to move to units the right:
x' = x + 2
y' = y
z' = z

This is translation.
I can also do rotation (upside down in the following case):

x' = -x
y' = -y
z' = z

Doing this one by one this is really boring...
I can do this instantaneoulsy... and simultanously... with matrixes.

Let's start by translataing rotaion into a matrix.
(I'm jumping ahead and say we want to get to:

Ex: matrix I.3 * [x,y,z] = [x',y',z']

But recall the rotation operation (-x and -y in the transformation)

This will translate into:
I.3' * [x,y,z] = [x',y',z']
I.3' where 1,1 is neg and 2,2 is also neg.

This  for rotaion is enough... But what if i want to invert it (rotation 180) and move 2 to the right...
Where in this I.3'do i have space to include the two? Nightmare ! Nowhere !!!

One solution for this is to use quaternions.
So now we would have something like:

I.4 * [x,y,z] = [x',y',z']

BUT -> algebra rule broken... I can multiply 4*4 matrix by 3*3...
So now we will use quaternions 
Our operations will become something like:

I.4 * [x,y,z,w] = [x',y',z', w']

Now note something... x, y, and z will be reserved for polinomial... linear transofrmation.
And w will be reserved for 0'type transformation -> translation !
So at all point we will have to use w=1 -> if this is not true we messed up somewhere !

-1 0 0 -2       x       x'
0 -1 0 0        y       y'
0 0 1 0         z       z'

The main from this for now is ->  to deal with geometric stuff we need 4*4 matrix !

There are different types of rendering
- Rasterisation
- Ray tracing

Rasterisation -> which is the fastes and simplist way to translate something into 2D convincing the guest to understand and perceive reality
Ray tracing -> tries to ensure the physics of law in the animation


How does ray tracing work:
- project millions of millions of light particles into an object and check how uch goes into the camer (in a very simples way)


How does rasterisation work?
- project (vertices) and paint (fill in with color). done 
Which means: send vertices into the GPU. This does the projection. 2D Cores.
Then we must color - shading. For the lattter we need not only the coordinates, but also the faces (to know where is light and dark, etc.)
This means that we only need two shaders (two programs that run in the GPU) - one for projeciton, the other for shading.
The name given was 'shader' because it was the first operation succeeded in the GPU.
Technically, you could call the programs as 'vertex shader' and 'fragment shader'.
