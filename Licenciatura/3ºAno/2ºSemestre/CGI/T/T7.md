## Problema: vetores de direção.

Posso fazer a multiplicação de pontos ou de vetores por uma matriz:

p' = M * p
n' = M * n

p -> point
n -> normal

Mas corro o risco de alterar o vetor normal do objeto erradamente...
Erradamente como? Tal que que a normal deixe de respeitar a direção da tangente do objeto:
    n * T = 0       => tem que se verificar sempre !
É uma propriedade que tem que se manter !

n' = M'*n
t' = M'*T
    sendo M' a tranformação que quero aplicar.

Existe uma forma que nos permite fazer esta transformação tal que se mantem a propriedade que queremos:
    n'*t' = 0

M' = (M^-1)T   => inverse and transpose

Qualquer matriz que tenha que se aplicar, não a uma posição, mas a uma direção, do this !!!

- nine slice (ninth for center)

Resumo por GPT:
- Texture Atlas (elements inside a single image)
    - A texture atlas is a single large image that contains many smaller textures packed together. Instead of uploading multiple small images to the GPU (which is expensive due to overhead and state changes), we upload one larger texture and reuse it by sampling different regions using texture coordinates (UVs).
    - This is efficient because the GPU can access all needed textures in one go, reducing draw calls and improving performance. By knowing the position of each sub-texture inside the atlas, we can map (“transform”) the correct portion onto objects, making rendering faster and more flexible.
    - Additionally, techniques like nine-slicing allow parts of a texture (e.g., corners, edges, center) to be reused and resized independently, which is especially useful for UI elements.
